가. chapter 04: 데이터베이스 모델링
    
            04-01: 데이터베이스 모델링과 데이터 모델의 개념
            
                1. 현실 세계에 존재하는 데이터를 컴퓨터 세계의 데이터베이스로 옮기는 변환 과정을 보통 데이터 모델링이라 한다.
                2. 현실 세계의 개채를 데이터베이스화 하는 작업을 추상화라고 한다.
                3. 총 3단계에 거쳐 데이터 모델링이 진행 되는데... 현실세계 -> 개념적 모델링을 통한 개념 세계 수립 -> 논리적 모델링을 통한 컴퓨터 데이터베이스에 저장
                4. 일반적으로 데이터 모델은 데이터 구조(data structure), 연산(operation), 제약조건(constraint)으로 구성되어 있다.
                
                
            04-02: 개체-관계 모델
            
                1. 현실 세계를 개체(entity)와 관계(relationship)으로 정의하며. 이를 개념적으로 모델링 하여 그림으로 표현 한 것을 E-R diagram이라 한다.
                2. 개체는 현실 세계에서 조직을 운영하는 데 꼭 필요한 사람이나 사물과 같이 구별 되는 모든 것을 의미한다.
                3. 개체 타입과 개체 인스턴스를 예를 들자면... 
                   3-1. 고객(이름, 주소, 연락처, 적립금)
                        위의 라인은 개체 타입이라고 부르며. 고객은 개체 이름.. 이름, 주소, 연락처, 적립금은 속성이라 불린다.
                        
                        <리철진, 성남시 분당구, 031-000-0000> 은 개체 인스턴스로 불리며. 비슷한 형식으로 인스턴스들이 쌓인 것을 개체 집합이라 정의한다.
                        
                        
                4. 개체는 다이아그램에서 보통 사각형으로 묘사한다. 
                5. 개체에는 속성들이 존재하는데 그 속성은 영어로 attribute라고 불리며. 한개의 개체에 여러 속성들이 존재 가능하다.
                6. 속성의 기준은 다음과 같다. 
                   6-1. 속성 값의 개수는 단일 값 속성과 다중 값 속성으로 구분 할 수 있다.
                        예시) 고객 아이디, 고객 명, 연락처... 이중 연락처는 핸드폰, 집전화, 이메일 등 값으로 구성 될 수 있음으로 연락처는 다중 값 속성이다. 
                   6-2. 의미의 분해 가능성은 단순 속성과 복합 소석으로 구분 할 수 있다.
                        예시) 단일 값 속성: 전화번호; 다중 값 속성: 생일(년, 월, 일) 
                   6-3. 기존 속성 값에서 유도 유도 속성으로 분류 할 수 있다.
                        예시) 값이 별도로 저장되는 것이 아니라 기존의 다른속성 값에서 유도되어 결정되는 속성을 유도 속성으로 분류한다.
                   6-4. 널 속성:
                        예시) 널 값이 허용되는 속성을 null attribute라고 한다.
                  
                   6-5. 키 속성: 모든 개체 인스턴스의 키 속성 값이 다르므로 키 속성은 대체 집합에 존재하는 각 개체 인스턴스들을 식별하는 데 사용된다.
                   
                 7. 관계: 관계는 개체와 개체가 맺고 있는 의미 있는 연관성으로, 개체-관계 모델의 중요한 요소다. 관계는 개체 집합들 사이의 대응 관계(correspondence, 즉 매핑mapping)을 의미한다. 관계를 여러 개체(타입) 사이에서 정의되는 관계 타입(relationship type)과 실제 속성 값으로 구성된 특정 개체 인스턴스들 간에 맺어진 실제 관계인 관계 인스턴스로 구분하여 표현하기도 한다. 고객 개체와 책 개체 사이에 정의된 구매 관계는 관계 타입이 된다. 
                 또한 관계도 개체처럼 속성을 가질 수 있다.
                 
                 8. 관계 유형에는... 일대일 관계, 일대다 관계, 다대다 관계가 있다.

             04-03: 논리적 데이터 모델
             
                1. 논리적 데이터 모델의 개념과 특성
                   -> E-R 다이어그램으로 표현된 개념적인 구조를 데이터베이스에 표현하는 형태를 결정하는 논리적 데이터 모델링에서는 데이터베이스 관리 시스템 종류가 중요하다.
                2. 계층 데이터 모델
                   -> 데이터베이스의 논리적 구조가 tree 형태다. 
                   
                3. 네크워크 데이터 모델
                   -> 데이터베이스의 논리적 구조가 그래프 또는 네트워크 형태이다.
