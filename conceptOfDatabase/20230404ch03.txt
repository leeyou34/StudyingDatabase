가. chapter 03: 데이터베이스 시스템
    
            03-01: 데이터베이스 시스템의 정의
            
                    1. 데이터베이스 시스템은 데이터베이스에 데이터를 저장하고, 저장된 데이터를 관리하여 조직에 필요한 정보를 생성해주는 시스템이다.
                    2-1. 데이터를 저장하고 저장된 데이터의 집합이 데이터베이스
                    2-2. 데이터베이스 관리 시스템은 데이터베이스에 저장된 데이터가 일관되고 무결한 상태로 유지되도록 관리하는 역할을 한다.
           
            03-02: 데이터베이스의 구조
            
                    1. 스키마: 데이터베이스에 저장되는 데이터 구조와 제약조건을 정의한 것이다. [ 고객번호 INT, 이름 CHAR(10), 나이 INT, 주소 CHAR(20) ]
                               고객번호는 정수, 이름은 문자열, 나이도 정수, 주소도 문자열만 허용하기로 했다면, 이 모든 정해진 내용이 스키마이다.
                               그리고 정의된 스키마에 따라 데이터에 실제로 저장된 값이 인스턴스(instance)이다.
                               
                    2. 3단계 데이터베이스 구조:  외부단계, 개념 단계, 내부 단계
                    
                    3. 데이터 독립성: 하나의 데이터 베이스에는 세 가지 유형의 스키마가 존재하지만, 각각의 스키마는 데이터베이스를 바라보는 관점이 다를 뿐 모두 같은 데이터베이스를 표현한다.
                                      3-1. 논리적 데이터 독립성
                                      3-2. 물리적 데이터 독립성
                    4. 데이터 사전: 데이터베이스는 조직 운영에 필요한 실제 데이터를 저장하는데, 저장된 데이터를 올바르게 관리하고 이용하려면 필요한 부가 정보도 저장해야 한다. 대표적인 부가 정보가 스키마와 사상 정보다.
                    
                    
            03-03: 데이터베이스 사용자
            
                    1. 데이터베이스 관리자: DBA
                       1-1. 데이터베이스 구성 요소 선정
                       1-2. 데이터베이스 스키마 정의
                       1-3. 물리적 저장 구조와 접근 방법 결정
                       1-4. 무결성 유지를 위한 제약조건 정의
                       1-5. 보안 및 접근 권한 정책 결정
                       1-6. 백업 및 회복 기법 정의
                       1-7. 시스템 데이터베이스 관리
                       1-8. 시스템 성능 감시 및 성능 분석
                       1-9. 데이터베이스 재구성
                       
                    2. 최종 사용자: 데이터를 조작(삽입, 삭제, 수정, 검색)하기 위해 데이터베이스에 접근하는 사람들을 일반 사용자 또는 최종 사용자(user)라고 한다.
                    
                    3. 응용 프로그래머 application programmer
                    
                    
             03-04: 데이터 언어
             
                    1. 데이터 언어는 크게 3가지로 정의된다. 데이터 정의어(Data Definition Language), 데이터 조작어(DML), 데이터 제어어(DCL)
                    2. 데이터 조작어에는 크게 두가지로 구분 된다.
                       2-1. 절차적 데이터 조작어: 구체적으로 알려주는 명령어
                       2-2. 비절차적 데이터 조작어: 단순한 지시를 입력한 명령어
                    3. 데이터 제어어 경우 다음과 같은 특성을 보장하기 위해 규칙이나 기법을 정의한다.
                       3-1. 무결성
                       3-2. 보안
                       3-3. 회복
                       3-4. 동시성
                       
             03-05: 데이터베이스 관리 시스템의 구성

            Practice questions are listed since they were pretty difficult.
            
            1. 스키마는 데이터 사전에 저장되고, 메타 데이터라고도 하며, 데이터베이스에 저장되는 데이터 구조와 제약조건에 대한 정의이다.
            
            2. 데이터베이스를 물리적 저장장치의 관점에서 이해한 구조다. 레코드의 구조, 레코드를 구성하는 필드 크기, 레코드의 물리적 순서, 인덱스를 이용한 레코드의 접근 경로 등과 같이 실제로 저장되는 방법을 정의한다. 이는 내부 스키마를 설명한다.
            
            3. 데이터베이스를 사용자 관점에서 이해한 구조다. 각 사용자에게 필요한 데이터베이스의 구조를 정의하여 하나의 데이터베이스에 여러 개가 존재할 수 있다. 3단계 데이터베이스 구조에서 설명과 관련있는 스키마는 외부 스키마이다.
            
            4. 개념 스키마에 대한 설명으로 옳은 것을 모두 고른다면. 조직 전체의 관점에서 본 데이터베이스의 구조이며 접근 권한 보안 정책을 정의 한다.
            
            5. 데이터베이스 관리 시스템이 필수 기능 중에서 데이터베이스의 논리적 구조와 물리적 구조 사이의 변환이 가능하도록 두 구조 사이의 대응 관계, 즉 사상Mapping을 명세하여 하나의 물리적 구조로 여러 사용자가 요구하는 데이터베이스의 구조를 지원하게 하는 것은 어떤 기능에 포함되는 것은 바로 사상 기능이다.
            
            6. 논리적 데이터의 독립성을 설명한 것은 개별 사용자나 응용 프로그램의 데이터 관점을 변경하지 않고 전체 데이터베이스의 논리적 구조를 변경시킬 수 있다.
            
            7. 물리적 데이터 독립성에 대한 설명으로 가장 적합한 것은 기존 응용 프로구램에 영향을 주지 않고 데이터베이스의 물리적 구조를 변경할 수 있는 것이다.
            
            8. 데이터 사전에 대한 설명을 하면.. 데이터 사전에 저장된 데이터를 메타 데이터라고 한다. 그리고 시스템 자신이 필요로 하는 스키마 및 여러 객체에 관한 정보를 저장하고, 사용자가 데이터 사전에 내용을 직접 추가하거나 수정 할 수 없다.
            
            9. 데이터 사전에 대한 설명으로 메타 데이터를 가지고 있는 시스템 데이터베이스로 시스템 카탈로그라고 한다.
            
            10. 데이터베이스 관리자는 데이터베이스 스키마 정의, 보안 및 접근 권한 정책 결정, 무결성 유지를 위한 제약조건 정의 업무를 수행한다.
            
            11. 사용자가 데이터의 삽입 삭제 수정 검색 등의 처리를 데이터베이스 관리 시스템에 요구하기 위해 사용하는 데이터 언어는 데이터 조작어라고 한다.
            
            12. 데이터 제어어의 기능을 몇가지 적자면... 무결성 유지, 회복 및 동시 공유 제어, 접근 제어 및 권한 부여가 있다.
            
            13. 데이터베이스의 스키마를 정의 변경 삭제할 수 있는 데이터 언어는 데이터 정의어라고 한다.
            
            14. 데이터의 보안, 무결성, 회복과 밀접한 관련이 있는 데이터 언어는 데이터 제어어라고 한다.
            
            15. 데이터베이스 관리 시스템의 역할에 대한 설명을 나열하자면... 트랜젝션 관리자는 무결성 제약조건 검사, 사용자의 접근 권한 검사, 병행 제어, 회복 등과 관련한 작업을 수행한다. 그리고 젖아 데이터 관리자는 디스크에 저장되어 있는 사용자 데이터베이스와 데이터 사전에 대한 접근을 책임진다. 마지막으로 DML 컴파일러는 데이터 조작어로 작성된 데이터의 처리 요구를 분석하여 런타임 데이터 베이스 처리기가 이해할 수 있도록 해석한다.
            
            16. 데이터베이스 관리 시스템의 주요 구성 요소에 대한 설명이다. DDL 컴파일러는 데이터베이스 관리자가 명세한 스키마 정의를 해석하고 데이터 사전에 저장하며, 런타임 데이터베이스 처리기는 저장 데이터 관리자를 통해 데이터베이스에 접근하여 데이터 처리 요구를 실제로 실행한다. 그리고 DML 프리 컴파일러는 응용 프로그램에 삽입된 데이터 조작어를 추출한다. 마지막으로 트랜잭션 관리자는 데이터베이스에 접근하는 과정에서 접근 권한 무결성 제약 조건 등을 검사한다.
            
            17. 스키마는 데이터베이스에 저장되는 데이터 구조와 제약조건을 정의한 것이다. 그리고 스키마에 따라 데이터베이스에 실제로 저장된 값을 인스턴스라고 한다.
            
            18. 3단계 데이터베이스 구조에서 개념/내부 사상은 개념 스키마와 내부 스키마의 대응 관계를 정의한 것으로, 저장 인터페이스라고 한다.
            
            19. 데이터 독립성은 하위 스키마를 변경하더라도 상위 스키마가 영향을 받지 않는 특성을 의미한다. 3단계 데이터베이스 구조에서는 두가지 유형으로 존재한다. 이 중 개념 스키마가 변경되더라도 외부 스키마가 영향을 받지 않는 것을 논리적 데이터 독립성이라고 한다.
            
            20. 데이터 사전은 데이터베이스에 저장되는 데이터에 관한 정보를 저장하는 곳으로, 스키마, 매핑 정보, 다양한 제약 조건 등을 저장한다. 그리고 데이터 사전에 저장되어 있는 정보에 실제로 접근하는 데 필요한 위치 정보는 데이터 디렉토리라는 곳에서 관리한다. 
            
            21. 데이터베이스 관리 시스템은 내부적으로 사용자의 데이터 처리 요구를 해석하여 처리하는 역할을 담당하는 질의 처리기와 디스크에 저장되어 있는 사용자 데이터베이스와 데이터 사전을 관리하고 접근하는 역할을 담당하는 저장데이터 관리자로 구성되어 있다.
            
            22. 데이터베이스의 스키마에 대해 간략한 설명을 하나 적자면. 데이터베이스에 저장되는 데이터 구조와 제약조건을 정의한다.
            
            23. 데이터베이스 시스템이란 데이터를 저장하고, 저장된 데이터를 관리하여 조직에 필요한 정보를 생성해주는 시스템이다.
            
            24. 데이터베이스 시스템의 주요 구성 요소 다섯 가지는 사용자, 데이터 언어, 데이터베이스 관리 시스템, 컴퓨터, 데이터베이스
            
            25. 데이터 독립성에 대한 의미란... 데이터 관리 시스템이 필요해서 이며 또한 하위 스키마를 변경해도 상위 스키마에 영향을 주지 않기 위함이라... 3단계 데이터베이스 구조에서 데이터 독립성을 실현하는 방법은 논리적 데이터 독십성 물리적 데이터 독립성으로 구성 되어있다.
            
            26. 데이터베이스 관리자가 담당하는 주요 업무는 데이터베이스 구성요소 선정, 데이터베이스 스키마 정의, 물리적 저장 구조와 접근 방법 결정, 마지막으로 백업 및 회복 방법 정의
            
            27. 데이터 언어를 사용 목적에 따라 3가지로 나뉜다. DDL, DML, DCL
