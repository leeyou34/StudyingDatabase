가. chapter 03: 데이터베이스 시스템
    
            03-01: 데이터베이스 시스템의 정의
            
                    1. 데이터베이스 시스템은 데이터베이스에 데이터를 저장하고, 저장된 데이터를 관리하여 조직에 필요한 정보를 생성해주는 시스템이다.
                    2-1. 데이터를 저장하고 저장된 데이터의 집합이 데이터베이스
                    2-2. 데이터베이스 관리 시스템은 데이터베이스에 저장된 데이터가 일관되고 무결한 상태로 유지되도록 관리하는 역할을 한다.
           
            03-02: 데이터베이스의 구조
            
                    1. 스키마: 데이터베이스에 저장되는 데이터 구조와 제약조건을 정의한 것이다. [ 고객번호 INT, 이름 CHAR(10), 나이 INT, 주소 CHAR(20) ]
                               고객번호는 정수, 이름은 문자열, 나이도 정수, 주소도 문자열만 허용하기로 했다면, 이 모든 정해진 내용이 스키마이다.
                               그리고 정의된 스키마에 따라 데이터에 실제로 저장된 값이 인스턴스(instance)이다.
                               
                    2. 3단계 데이터베이스 구조:  외부단계, 개념 단계, 내부 단계
                    
                    3. 데이터 독립성: 하나의 데이터 베이스에는 세 가지 유형의 스키마가 존재하지만, 각각의 스키마는 데이터베이스를 바라보는 관점이 다를 뿐 모두 같은 데이터베이스를 표현한다.
                                      3-1. 논리적 데이터 독립성
                                      3-2. 물리적 데이터 독립성
                    4. 데이터 사전: 데이터베이스는 조직 운영에 필요한 실제 데이터를 저장하는데, 저장된 데이터를 올바르게 관리하고 이용하려면 필요한 부가 정보도 저장해야 한다. 대표적인 부가 정보가 스키마와 사상 정보다.
                    
                    
            03-03: 데이터베이스 사용자
            
                    1. 데이터베이스 관리자: DBA
                       1-1. 데이터베이스 구성 요소 선정
                       1-2. 데이터베이스 스키마 정의
                       1-3. 물리적 저장 구조와 접근 방법 결정
                       1-4. 무결성 유지를 위한 제약조건 정의
                       1-5. 보안 및 접근 권한 정책 결정
                       1-6. 백업 및 회복 기법 정의
                       1-7. 시스템 데이터베이스 관리
                       1-8. 시스템 성능 감시 및 성능 분석
                       1-9. 데이터베이스 재구성
                       
                    2. 최종 사용자: 데이터를 조작(삽입, 삭제, 수정, 검색)하기 위해 데이터베이스에 접근하는 사람들을 일반 사용자 또는 최종 사용자(user)라고 한다.
                    
                    3. 응용 프로그래머 application programmer
                    
                    
             03-04: 데이터 언어
             
                    1. 데이터 언어는 크게 3가지로 정의된다. 데이터 정의어(Data Definition Language), 데이터 조작어(DML), 데이터 제어어(DCL)
                    2. 데이터 조작어에는 크게 두가지로 구분 된다.
                       2-1. 절차적 데이터 조작어: 구체적으로 알려주는 명령어
                       2-2. 비절차적 데이터 조작어: 단순한 지시를 입력한 명령어
                    3. 데이터 제어어 경우 다음과 같은 특성을 보장하기 위해 규칙이나 기법을 정의한다.
                       3-1. 무결성
                       3-2. 보안
                       3-3. 회복
                       3-4. 동시성
