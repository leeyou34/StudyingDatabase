가. chapter 02: 데이터베이스 관리 시스템
    
            02-01: 데이터베이스 관리 시스템의 등장 배경
            
                  1. 파일 시스템으로 부터 관리된 데이터에는 같은 내용의 데이터가 여러 파일에 중복 저장 되었고
                  2. 으용 프로그램이 데이터 파일에 종속적이였으며
                  3. 데이터 파일에 대한 동시 공유, 보안, 회복 기능이 부족했고
                  4. 최종적으로 응용 프로그램을 개발하기 쉽지 않았다.
                  
    
            02-02: 데이터베이스 관리 시스템의 정의
    
                  1. 데이터베이스 관리 시스템은 파일 시스템의 데이터 중복과 데이터 종속 문제를 해결하기 위해 제시된 소프트웨어이다.
                  
                  데이터베이스 관리 시스템에서의 데이터 관리 예시) 
                  사용자 <-> 고객 관리 응용 프로그램, 주문 관리 응용 프로그램 <-> 데이터베이스 관리 시스템 <-> 데이터베이스
                  
                  2. 데이터베이스 관리 시스템이 제공하는 주요 기능은 다음 세가지로 요약된다.
                    2-1) 정의 기능: 데이터베이스 구조를 정의하거나 수정할 수 있다.
                    2-2) 조작 기능: 데이터를 삽입, 삭제, 수정, 검색하는 연산을 할 수 있다.
                    2-3) 제어 기능: 데이터를 항상 정확하고 안전하게 유지 가능하다.
                    
    
            02-03: 데이터베이스 관리 시스템의 장,단점
            
                  1. 장점
                    1-1) 데이터 중복 통제 가능
                    1-2) 데이터 독립성이 확보
                    1-3) 데이터 동시 공유 가능
                    1-4) 데이터 보안 향상
                    1-5) 데이터 무결성 유지(keeping the accuracy)
                    1-6) 데이터 표준화 가능
                    1-7) 데이터 장애 발생시 회복 가능
                    1-8) 응용 프로그램 개발 비용 절감
                    
                  2. 단점
                    2-1) 고비용: 관리 시스템을 따로 설치함으로서 구매 비용이 많이 들며, 동시 접속 사용자 수에 따라 제품 가격 증가
                    2-2) 백업과 회복 방법이 복잡
                    2-3) 중앙 집중 관리로 인한 취약점이 존재한다.
              
              
              02-04: 데이터베이스 관리 시스템의 발전 과정
              
                  1. 1세대 데이터베이스 관리 시스템: 네트워크: 계층 DBMS 
                  2. 2세대 데이터베이스 관리 시스템: 관계 DBMS
                  3. 3세대 데이터베이스 관리 시스템: 객체 지향* 객체관계 DBMS
                  4. 4세대 데이터베이스 관리 시스템: NoSQL, NewSQL DBMS
